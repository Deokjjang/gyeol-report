# 토스페이먼츠 결제경로 캡처 체크리스트

이 문서는 토스페이먼츠 결제경로 파일을 만들기 전에 결리포트의 화면 흐름을 확인하기 위한 체크리스트다. 실제 PPT 제작 파일은 별도로 작성한다.

## 캡처 순서

1. 메인 페이지
2. 상품 목록 페이지
3. 상품 상세 페이지
4. 리포트 입력 페이지
5. 입력값 완료 상태
6. 결제 직전 확인
7. 필수 체크박스 전체 체크 후 결제 버튼 활성화 상태
8. 토스 결제창
9. 결제 성공 화면
10. 리포트 결과 화면
11. 사업자정보 및 정책 링크 확인 화면

## 각 화면 URL

| 화면 | 로컬 URL | 확인 포인트 |
| --- | --- | --- |
| 메인 페이지 | `http://localhost:3000/` | 결제 가능한 상품과 시작 버튼 |
| 상품 목록 페이지 | `http://localhost:3000/products` | 사주×MBTI 종합 리포트 상품 노출 |
| 상품 상세 페이지 | `http://localhost:3000/products/saju-mbti-full` | 상품명, 가격, 제공 방식, 정책 링크 |
| 리포트 입력 페이지 | `http://localhost:3000/report/new` | 이름, 생년월일, 출생시간, 성별, MBTI 입력 |
| 결제 직전 확인 | `http://localhost:3000/report/new` | 입력값 최종 확인과 거래조건 고지 |
| 토스 결제창 | 결제 버튼 클릭 후 표시 | 토스 결제창 진입 화면 |
| 결제 성공 화면 | `http://localhost:3000/payments/toss/success` | 결제 승인 및 리포트 보기 동선 |
| 리포트 결과 화면 | `http://localhost:3000/reports/<report_id>` | 리포트 본문 또는 리포트 준비 완료 상태 |
| 사업자정보 | `http://localhost:3000/business` | 상호명, 대표자, 사업자등록번호, 고객센터 |
| 이용약관 | `http://localhost:3000/terms` | 서비스 이용 조건 |
| 개인정보처리방침 | `http://localhost:3000/privacy` | 개인정보 처리 기준 |
| 환불정책 | `http://localhost:3000/refund` | 환불 및 재생성 기준 |

법무 페이지 경로도 함께 확인할 수 있다.

- `http://localhost:3000/legal/business-info`
- `http://localhost:3000/legal/terms`
- `http://localhost:3000/legal/privacy`
- `http://localhost:3000/legal/refund`

## PPT 구성 권장 순서

1. 메인 페이지 전체 화면
2. 상품 목록에서 결제 가능 상품 확인
3. 상품 상세 페이지의 상품명, 가격, 제공 방식
4. 입력 페이지의 입력 항목
5. 입력값 완료 상태
6. 결제 직전 확인 화면
7. 필수 체크박스 전체 체크와 활성화된 결제 버튼
8. 토스 결제창
9. 결제 성공 화면과 리포트 보기 버튼
10. 리포트 결과 화면
11. 푸터의 사업자정보
12. 이용약관, 개인정보처리방침, 환불정책, 사업자정보 페이지

## 주의사항

- 화면 캡처는 실제 사용자에게 보이는 production-like 화면 기준으로 준비한다.
- 캡처 화면에 개발자 도구, 콘솔, 내부 API 응답, 환경값이 보이지 않게 한다.
- 입력값은 예시 사용자 정보로 준비하되, 주민등록번호나 불필요한 민감 값을 입력하지 않는다.
- 결제 직전 확인 화면에서 입력값, 상품명, 가격, 제공 방식, 환불 안내, 정책 링크가 한 번에 확인되도록 캡처한다.
- 성공 화면에서는 리포트 보기 동선이 보이도록 캡처한다.
- 결과 화면은 생성된 리포트가 있으면 본문을, 없으면 리포트 준비 완료 상태를 캡처한다.
