2. Visible Element Counting

Visible elements count only:

heavenly stems + earthly branches main element

Use:

STEM_ELEMENT
BRANCH_MAIN_ELEMENT

Rule

Each visible stem/branch counts as 1.

For full 사주 with 4 pillars:

4 stems + 4 branches = 8 visible units

For 삼주 without hour:

3 stems + 3 branches = 6 visible units

⸻

3. Weighted Element Counting

Weighted elements count:

heavenly stems
+ earthly branch main element
+ hidden stems

Use:

STEM_ELEMENT
BRANCH_MAIN_ELEMENT
HIDDEN_STEMS
ELEMENT_WEIGHTS

Rule

For each pillar:

* Stem element adds ELEMENT_WEIGHTS.heavenlyStem
* Branch main element adds ELEMENT_WEIGHTS.earthlyBranchMain
* Hidden stems add their own configured weight from HIDDEN_STEMS

Do not hardcode hidden stem weights in this module.
Use constants.

⸻

4. Element Labeling

Implement:

export function getElementLabels(
  weighted: Record<FiveElement, number>,
): ElementLabel[];

V1 Thresholds

Use simple deterministic thresholds for now:

MISSING = 0
WEAK = > 0 and < 1.5
STRONG = >= 3.0

Label Rules

For each element:

* if value === 0 → ELEMENT_MISSING
* else if value < 1.5 → ELEMENT_WEAK
* else if value >= 3.0 → ELEMENT_STRONG
* else → no label

Examples:

WOOD 0      -> WOOD_MISSING
WATER 1.0   -> WATER_WEAK
EARTH 3.2   -> EARTH_STRONG
FIRE 2.0    -> no label

Note

These thresholds are V1 deterministic heuristics.
They may be adjusted later and must be versioned if changed.

⸻

5. Yin/Yang Counting

Count yin/yang from visible stems and visible branches only.

Use:

STEM_YIN_YANG
BRANCH_YIN_YANG

Rule

* Each visible stem counts 1.
* Each visible branch counts 1.
* Hidden stems do not count for V1 yin/yang balance.

Label Threshold

For full 사주:

yin + yang = 8

For 삼주:

yin + yang = 6

Use ratio:

yinRatio = yin / total
yangRatio = yang / total

Labels:

YIN_HEAVY if yinRatio >= 0.65
YANG_HEAVY if yangRatio >= 0.65
BALANCED otherwise

7. Ten Gods Logic

Ten Gods are calculated by relation between:

dayMaster element/yin-yang
targetStem element/yin-yang

Use:

STEM_ELEMENT
STEM_YIN_YANG

Five Element Generation / Control

Use the standard cycle:

Generates:
WOOD -> FIRE
FIRE -> EARTH
EARTH -> METAL
METAL -> WATER
WATER -> WOOD
Controls:
WOOD -> EARTH
EARTH -> WATER
WATER -> FIRE
FIRE -> METAL
METAL -> WOOD

⸻

8. Ten Gods Mapping Rule

Let:

self = dayMaster
target = targetStem

Same element

same yin/yang     -> 比肩
different yin/yang -> 劫財

Self generates target

same yin/yang      -> 食神
different yin/yang -> 傷官

Target generates self

same yin/yang      -> 偏印
different yin/yang -> 正印

Self controls target

same yin/yang      -> 偏財
different yin/yang -> 正財

Target controls self

same yin/yang      -> 偏官
different yin/yang -> 正官

Important: Do not reverse 正/偏 polarity casually.
Use this exact mapping for V1.

9. Stem Ten Gods

Implement:

getStemTenGods(dayMaster, pillars)

Rule

* year stem → year Ten God
* month stem → month Ten God
* hour stem → hour Ten God if hour exists
* day stem is the day master itself, so exclude day stem from result

Example output:

{
  year: "正財",
  month: "偏官",
  hour: "食神"
}

⸻

10. Hidden Stem Ten Gods

Implement:

getHiddenStemTenGodEntries(dayMaster, pillars)

Use:

HIDDEN_STEMS

For each branch in year/month/day/hour:

* Get hidden stems.
* Calculate Ten God for each hidden stem.
* Preserve hidden stem weight.

Include day branch hidden stems too.

Do not exclude day branch.

⸻

11. Ten God Distribution

Implement:

getTenGodDistribution(dayMaster, pillars)

Initial empty distribution

All Ten Gods must exist with 0.

Count

* Visible non-day stems count 1.0.
* Hidden stems count their hidden stem weight.

Do not count day stem as Ten God.

Example

If year stem is 正財, add 1.0 to 正財.
If hidden stem 癸 has weight 0.3 and maps to 正印, add 0.3 to 正印.
