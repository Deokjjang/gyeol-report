# Task 14 — Report Validation Filter
## Context
We are building **결리포트 / Gyeol Report**, a mobile-first Saju × MBTI self-narrative report service.
Task 13 created:
- Report output types
- Prompt builder
- LLM generation wrapper
- Minimal generated report shape validation
This task implements the **report validation filter** that checks generated report text before it can be saved or shown to users.
V1 rule:
```txt
LLM output must pass validation before use.

Do not implement API routes, UI, payment, Supabase writes, or real LLM provider integration in this task.

Relevant SSOT:

* docs/ssot/03_REPORT_SPEC.md
* docs/ssot/04_TECH_SPEC.md
* docs/ssot/05_DEVELOPMENT_PLAN.md

⸻

Goal

Create deterministic validation for GeneratedReport.

Validation must catch:

1. Missing required sections
2. Empty sections
3. Forbidden direct terms
4. Forbidden fate/health/legal/investment/marriage certainty patterns
5. V1 excluded feature mentions
6. MBTI/Saju direct equivalence claims
7. Length range warnings

1. Create validateReport.ts

Create:

src/lib/analysis/validateReport.ts

Export:

import type { GeneratedReport } from "./reportTypes";
export type ReportValidationSeverity = "ERROR" | "WARNING";
export type ReportValidationIssue = {
  severity: ReportValidationSeverity;
  code: string;
  message: string;
  path?: string;
};
export type ReportValidationResult = {
  ok: boolean;
  issues: ReportValidationIssue[];
};
export function validateGeneratedReport(
  report: GeneratedReport,
): ReportValidationResult;
export function assertValidGeneratedReport(report: GeneratedReport): void;

8. Length Validation

Length issues are warnings, not errors.

Preview

Target:

500~800 Korean characters

If preview length < 300:

PREVIEW_TOO_SHORT

If preview length > 1000:

PREVIEW_TOO_LONG

Full Report Total

Target:

3,500~5,000 Korean characters

If total full report length < 2500:

FULL_REPORT_TOO_SHORT

If total full report length > 6500:

FULL_REPORT_TOO_LONG

Severity

WARNING

Length warnings do not make ok false.

10. ok Rule

ok = no issues with severity ERROR

Warnings do not fail validation.

11. assertValidGeneratedReport(report)

Behavior:

* Calls validateGeneratedReport(report).
* If ok === false, throw:

Generated report failed validation.

Include issue codes in the error message if practical.

Example:

Generated report failed validation: FORBIDDEN_DIRECT_TERM, MISSING_SECTION
