2. Hidden Stem Extraction Rules

Use existing constant:

HIDDEN_STEMS

getHiddenStemsForBranch(branch)

Behavior:

* Return hidden stems for the given branch.
* Include:
    * branch
    * stem
    * weight
    * kind
* Must not mutate constants.
* Return readonly-compatible data.

Example:

getHiddenStemsForBranch("丑")

Expected:

[
  { branch: "丑", stem: "己", kind: "MAIN", weight: 0.6 },
  { branch: "丑", stem: "癸", kind: "SUB", weight: 0.3 },
  { branch: "丑", stem: "辛", kind: "MINOR", weight: 0.1 }
]

getHiddenStemsForPillars(pillars)

Behavior:

* Extract hidden stems for year branch.
* Extract hidden stems for month branch.
* Extract hidden stems for day branch.
* Extract hidden stems for hour branch if hour exists.
* Preserve order:

year → month → day → hour

5. Relation Detection Rules

Pair Presence

A relation exists if both members of a pair are present in the input list.

Example:

findStemCombinations(["甲", "丙", "己"])

Expected:

[{ stems: ["甲", "己"] }]

Duplicate Handling

If duplicates exist, do not return duplicate relations.

Example:

findBranchClashes(["子", "子", "午"])

Expected:

[{ branches: ["子", "午"] }]

Order

Return relations in the order of the constants table.

Do not sort by input order.

6. getRelationsForPillars(pillars)

Behavior:

1. Collect stems from:
    * year stem
    * month stem
    * day stem
    * hour stem if exists
2. Collect branches from:
    * year branch
    * month branch
    * day branch
    * hour branch if exists
3. Return:

{
  stemCombinations,
  branchCombinations,
  branchClashes
}

For 삼주 without hour, only use year/month/day.
