Task 12 — Interpretation Block Model and Seed

# Task 12 — Interpretation Block Model and Seed
## Context
We are building **결리포트 / Gyeol Report**, a mobile-first Saju × MBTI self-narrative report service.
Task 11 implemented deterministic Saju-MBTI comparison.
This task creates the interpretation block model and initial seed content used by report generation.
V1 report generation must be block-based, not free-form LLM guessing.
The LLM will later act only as an editor that connects selected blocks into natural Korean prose.
Do not implement LLM generation, API routes, UI, payment, or final report generation in this task.
Relevant SSOT:
- `docs/ssot/03_REPORT_SPEC.md`
- `docs/ssot/04_TECH_SPEC.md`
- `docs/ssot/05_DEVELOPMENT_PLAN.md`

7. Combined Tag Set

Inside selectInterpretationBlocks, combine:

analysisTags.mbti.type
analysisTags.mbti.traits
analysisTags.saju.structureTags
analysisTags.overlapTags
analysisTags.conflictTags
analysisTags.summarySignals

Into a Set.

This combined tag set is used to match block conditions.

⸻

8. Block Match Logic

A block matches if:

block.active === true
AND every condition in block.conditions exists in combined tag set

If conditions is empty, the block is generic fallback.

10. Selection Per Section

For each section:

1. Collect blocks whose category is allowed by the section.
2. Filter matching blocks.
3. Sort by:
    * higher weight first
    * then lexicographic id
4. Take max blocks per section.

Recommended max:

const SECTION_MAX_BLOCKS: Record<ReportSectionKey, number> = {
  narrativeImage: 1,
  coreTemperament: 4,
  sajuStructure: 5,
  mbtiOverlap: 4,
  mbtiConflict: 4,
  loveStyle: 3,
  workStudyStyle: 3,
  careerDirection: 3,
  moneySense: 3,
  relationshipPattern: 3,
  repeatingWeakness: 3,
  compensationStrategy: 3,
};

11. Fallback Behavior

Every section must have at least one block.

If no condition-specific block matches:

1. Use generic fallback blocks in that section category.
2. Fallback block should have conditions: [].
3. If still empty, return empty array but tests should ensure initial seed prevents this.

Required generic fallback blocks:

NARRATIVE_IMAGE
LOVE_STYLE
WORK_STYLE
CAREER_DIRECTION
MONEY_SENSE
RELATIONSHIP_PATTERN
WEAKNESS_PATTERN
COMPENSATION_STRATEGY

Preferred fallback for core sections too.

⸻

12. Duplicate Meaning Rule

For V1, duplicate meaning removal is simple:

* Do not select duplicate block IDs.
* If two blocks have same key, keep higher weight.
* For same key and same weight, keep lexicographically smaller ID.
