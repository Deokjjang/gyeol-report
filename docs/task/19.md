2. Toss Client

Create:

src/lib/payments/tossClient.ts

Export:

import "server-only";
export type TossConfirmInput = {
  paymentKey: string;
  orderId: string;
  amount: number;
};
export type TossConfirmResult = {
  paymentKey: string;
  orderId: string;
  totalAmount: number;
  status: string;
  raw: unknown;
};
export type TossClient = {
  confirmPayment(input: TossConfirmInput): Promise<TossConfirmResult>;
};
export function createTossClient(): TossClient;

Behavior

Use Toss secret key:

TOSS_SECRET_KEY

If missing, throw:

Missing TOSS_SECRET_KEY

Confirm API

Call Toss Payments confirm endpoint.

Use Basic Auth with secret key.

Implementation should follow Toss API expectations:

POST https://api.tosspayments.com/v1/payments/confirm
Authorization: Basic base64(TOSS_SECRET_KEY + ":")
Content-Type: application/json

Body:

{
  "paymentKey": "...",
  "orderId": "...",
  "amount": 990
}

Response

Return:

{
  paymentKey,
  orderId,
  totalAmount,
  status,
  raw
}

Error Handling

If Toss API fails:

* throw a clear error
* include Toss error message if available
* do not expose secret key

Example:

Toss payment confirmation failed.

4. Confirm Behavior

Validate Input

If missing paymentKey:

Missing paymentKey.

If missing orderId:

Missing orderId.

If invalid amount:

Invalid amount.

⸻

5. Find Payment

Use:

db.findPaymentByOrderId(orderId)

If null:

Payment not found.

⸻

6. Status Checks

If payment already PAID:

Return idempotent success:

{
  resultToken,
  fullReportUrl: `${appUrl}/report/${resultToken}`
}

Do not call Toss again.

If payment status is not READY:

Payment is not ready for confirmation.

If report payment status is already PAID, also return idempotent success.

⸻

7. Amount Verification

Critical.

Compare:

input.amount === payment.amount

If mismatch:

1. Mark payment failed.
2. Insert PAYMENT_FAILED event.
3. Throw:

Payment amount mismatch.

Never call Toss confirm if amount mismatches.

⸻

8. Confirm with Toss

Call:

tossClient.confirmPayment({
  paymentKey,
  orderId,
  amount,
})

Then verify:

result.orderId === payment.orderId
result.totalAmount === payment.amount

If mismatch:

1. Mark payment failed.
2. Insert PAYMENT_FAILED event.
3. Throw:

Toss confirmation response mismatch.

⸻

9. Success DB Updates

On success:

1. markPaymentPaid
2. markReportPaid
3. insertReportEvent(PAYMENT_CONFIRMED)

markPaymentPaid should update

payments.status = PAID
payments.payment_key = paymentKey
payments.raw_response = Toss raw response
payments.paid_at = now()

markReportPaid should update

reports.payment_status = PAID
reports.paid_at = now()
reports.updated_at = now()

PAYMENT_CONFIRMED payload

{
  "provider": "TOSS",
  "orderId": "...",
  "paymentKey": "...",
  "amount": 990,
  "currency": "KRW"
}

⸻

10. Toss Failure Handling

If Toss client throws:

1. markPaymentFailed
2. insertReportEvent(PAYMENT_FAILED)
3. rethrow generic error:

Toss payment confirmation failed.

Do not mark report paid.

11. API Route

Create:

src/app/api/payments/toss/confirm/route.ts

Export:

export async function POST(request: Request): Promise<Response>

Request body

{
  "paymentKey": "...",
  "orderId": "...",
  "amount": 990
}

Behavior

1. Parse JSON.
2. Create Supabase-backed DB adapter.
3. Create Toss client.
4. Call confirmTossPayment().
5. Return:

{
  "resultToken": "...",
  "fullReportUrl": "http://localhost:3000/report/..."
}

Status codes

200: success
400: validation / amount mismatch / invalid status
404: payment not found
500: unexpected or Toss confirm failure

Do not leak stack traces.
