4. Solar DateTime Construction

For calendarType = SOLAR, construct KST datetime:

If birth time exists

YYYY-MM-DDTHH:mm:00+09:00

Example:

2024-02-04 + 17:27
→ 2024-02-04T17:27:00+09:00

If birth time unknown

Use noon for year/month solar term boundary calculation:

YYYY-MM-DDT12:00:00+09:00

Reason:

* Year/month pillar depends on solar-term boundary time.
* Without birth time, exact boundary-day interpretation can be uncertain.
* Noon is deterministic for V1.

Add notice if birth time unknown:

출생시간을 모르는 상태이므로 이 리포트는 년주·월주·일주 중심으로 해석합니다. 시주가 포함되지 않아 일부 내면 표현, 말년 흐름, 세부 성향 해석은 제한될 수 있습니다.

⸻

5. Boundary Day Notice

If birthTimeUnknown === true, and birth date is on the same KST date as any solar-term boundary relevant to year/month calculation, add notice:

출생시간을 모르는 상태에서 절기 경계일에 해당하므로 년주 또는 월주가 달라질 수 있습니다.

Implementation can be simple:

* Get current year solar term data.
* Check whether any boundary isoDateTimeKst.slice(0, 10) === birthDate.
* If yes, add notice.
* If previous year data is needed for January dates, check previous/current year safely if available.

If solar term data is unsupported, allow existing year/month calculation to throw.

⸻

6. 23시대 Notice

If birth time is provided and hour is 23, add notice:

23시대 출생자는 명리학 관점에 따라 일주 계산 기준이 달라질 수 있습니다. 결리포트 V1은 기본적으로 00:00 기준 일자 변경 방식을 사용합니다.

Do not change day pillar for 23:00 in V1.
