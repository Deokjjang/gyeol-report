4. Order ID Utility

Inside prepareTossPayment.ts, create private function:

function createOrderId(): string;

Requirements:

* URL-safe
* short enough for Toss
* unique enough
* no Math.random

Recommended:

import crypto from "crypto";
function createOrderId(): string {
  return `gyeol_${crypto.randomBytes(12).toString("hex")}`;
}

5. Customer Key Policy

For no-login V1, create deterministic customer key from result token.

Do not use raw token directly.

Private function:

function createCustomerKey(resultToken: string): string;

Recommended:

import crypto from "crypto";
function createCustomerKey(resultToken: string): string {
  return `guest_${crypto
    .createHash("sha256")
    .update(resultToken)
    .digest("hex")
    .slice(0, 32)}`;
}

7. Prepare Payment Behavior

Validate token

If empty:

Invalid result token.

Find report

If not found:

Report not found.

Generation status checks

If report is not ready:

* PENDING or GENERATING:

Report is not ready for payment yet.

* FAILED:

Report generation failed.

Payment status checks

If already paid:

Report is already paid.

If refunded:

Report was refunded.

Price

Use:

getCurrentReportPrice(now)

Create payment

Insert row:

provider = TOSS
order_id
amount
currency = KRW
status = READY
raw_request

Event

Insert:

PAYMENT_READY

Payload:

{
  "provider": "TOSS",
  "orderId": "...",
  "amount": 990,
  "currency": "KRW"
}

Return

{
  orderId,
  amount,
  currency: "KRW",
  orderName: "결리포트 전체 리포트",
  customerKey,
  tossClientKey
