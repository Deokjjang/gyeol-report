3. Normalization Rules

Implement:

export function normalizeMbtiType(value: string): MbtiType;

Behavior:

"entj" -> "ENTJ"
" ENTJ " -> "ENTJ"
"unknown" -> "UNKNOWN"
"" -> "UNKNOWN"
invalid string -> "UNKNOWN"

Do not throw for invalid user MBTI input.
Return "UNKNOWN".

⸻

4. getMbtiProfile(type)

Behavior:

* Return profile for valid MbtiType.
* Return UNKNOWN profile if type is invalid by runtime guard.

Even though TypeScript type is constrained, runtime should be safe.

⸻

5. getMbtiTraitTags(type)

Behavior:

* Normalize type if needed internally.
* Return profile traits.
* For UNKNOWN, return empty array or minimal neutral traits.

Recommended:

UNKNOWN.traits = []

⸻

6. MBTI Profile Content Rules

Each 16-type profile must include:

type
axes
traits
strengths
weaknesses
relationshipStyle
workStyle
decisionStyle
description

Content Style

Descriptions must be:

* Short
* Non-diagnostic
* Self-perception oriented
* Suitable for later report generation
* Not overly flattering
* Not deterministic

Forbidden

Do not write:

ENTJ is born leader.
INTJ always succeeds.
INFP is weak.
ESTP cannot be serious.
This type is better than others.

Use:

ENTJ often identifies with direction-setting, efficiency, and decisive execution.
