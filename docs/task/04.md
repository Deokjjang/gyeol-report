# Task 04 — Year and Month Pillar Calculation
## Context
We are building **결리포트 / Gyeol Report**, a mobile-first Saju × MBTI self-narrative report service.
Task 03 implemented deterministic day pillar calculation.
This task implements **year pillar and month pillar calculation only** using V1 solar-term policy.
Do not implement hour pillar, lunar conversion, five elements, ten gods, hidden stems, relations, report generation, API routes, or UI in this task.
Relevant SSOT:
- `docs/ssot/02_SAJU_CALC_SPEC.md`
- `docs/ssot/05_DEVELOPMENT_PLAN.md`
## Goal
Implement deterministic year pillar and month pillar calculation for a solar date under the V1 policy:
```txt
Year pillar = 입춘 기준
Month pillar = 절기 기준 월령
Timezone = Asia/Seoul

Important Implementation Note

For this task, use a V1 deterministic solar-term table rather than astronomical calculation.

6. Implement getActiveSolarTermBoundary(solarDateTimeKst)

Behavior:

* Parse year from input.
* Load current year data.
* Find the latest boundary whose isoDateTimeKst <= solarDateTimeKst.
* If none found in current year, load previous year data and use its DAESEOL/SOHAN logic as appropriate.

Simpler acceptable V1 behavior:

* Combine previous year + current year boundaries.
* Sort by ISO datetime.
* Pick latest boundary <= input.
* Throw if previous year data missing.

This is important for dates before 입춘 and before 소한.

9. Year Pillar Logic

Use 입춘 boundary.

Behavior:

if input is before IPCHUN of input year:
  sajuYear = input year - 1
else:
  sajuYear = input year

Then calculate year pillar.

Year Pillar Formula

Use known reference:

1984 = 甲子 year

Formula:

cycleIndex = positiveModulo(sajuYear - 1984, 60)
yearPillar = SEXAGENARY_CYCLE[cycleIndex]

Examples:

2024 after 입춘 = 甲辰
2024 before 입춘 = 癸卯
1984 after 입춘 = 甲子
1984 before 입춘 = 癸亥

⸻

10. Month Pillar Logic

Use active solar-term boundary.

1. Find active boundary.
2. Its monthBranch determines month branch.
3. Determine month stem from year stem and month index.

Month Branch Order

Use existing constant:

MONTH_BRANCHES_BY_SOLAR_TERM

Order:

寅 卯 辰 巳 午 未 申 酉 戌 亥 子 丑

Month Stem Formula

Use existing constant:

MONTH_STEM_START_BY_YEAR_STEM

This gives the stem for 寅 month.

Then:

monthIndex = index of active month branch in MONTH_BRANCHES_BY_SOLAR_TERM
startStem = MONTH_STEM_START_BY_YEAR_STEM[yearStem]
startStemIndex = STEM_INDEX[startStem]
monthStem = HEAVENLY_STEMS[(startStemIndex + monthIndex) % 10]

Return:

{ stem: monthStem, branch: activeBoundary.monthBranch }

⸻

11. Important Month-Year Stem Basis

Month stem uses the Saju year stem, not necessarily Gregorian input year stem.

So getMonthPillarFromSolarDateTime() must derive the year pillar first with 입춘 policy and use its stem.
