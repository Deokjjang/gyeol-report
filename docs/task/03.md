We are building **결리포트 / Gyeol Report**, a mobile-first Saju × MBTI self-narrative report service.

## Goal
Implement deterministic day pillar calculation from a solar date using a fixed epoch and the 60갑자 cycle.
The same input date must always return the same day pillar.

2. Fixed Epoch Policy

Use one fixed known day-pillar epoch.

For implementation, use:

1984-02-02 = 甲子 day

This is a commonly used 甲子 reference date for sexagenary day-cycle calculation.

Required Constants

Inside pillars.ts, define:

const DAY_PILLAR_EPOCH_DATE = "1984-02-02";
const DAY_PILLAR_EPOCH_INDEX = 0;

Where:

SEXAGENARY_CYCLE[0] = 甲子

Calculation

daysDiff = difference in UTC calendar days between input date and epoch date
cycleIndex = positiveModulo(DAY_PILLAR_EPOCH_INDEX + daysDiff, 60)
dayPillar = SEXAGENARY_CYCLE[cycleIndex]

⸻

3. Date Parsing

Implement strict date parsing.

Requirements

* Accept only YYYY-MM-DD.
* Reject invalid format.
* Reject invalid dates such as 2024-02-30.
* Use UTC date construction to avoid local timezone drift.
* Do not use JavaScript new Date("YYYY-MM-DD") directly without validation.

Recommended helper:

type ParsedDate = {
  year: number;
  month: number;
  day: number;
};
function parseIsoDateStrict(value: string): ParsedDate {
  // validate YYYY-MM-DD
  // validate actual calendar date
}

Then:

function toUtcDateMs(date: ParsedDate): number {
  return Date.UTC(date.year, date.month - 1, date.day);
}

⸻

4. Positive Modulo

Implement:

function positiveModulo(value: number, modulo: number): number {
  return ((value % modulo) + modulo) % modulo;
}

This is required so dates before the epoch work correctly.

⸻

5. Date Difference

Use exact UTC day difference.

const MS_PER_DAY = 24 * 60 * 60 * 1000;
const diffDays = Math.round((targetMs - epochMs) / MS_PER_DAY);

Because both dates are created by Date.UTC(year, month - 1, day), the result should be an integer.
