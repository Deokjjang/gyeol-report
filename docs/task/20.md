2. Full Report Fetcher

Create:

src/lib/reports/getFullReport.ts

Export:

import type { FullReport } from "@/lib/analysis/reportTypes";
import type { FullReportResponse } from "./fullReportTypes";
export type FullReportDbPort = {
  findFullReportByToken(token: string): Promise<{
    resultToken: string;
    paymentStatus: "UNPAID" | "PAID" | "REFUNDED";
    generationStatus: "PENDING" | "GENERATING" | "READY" | "FAILED";
    fullReport: unknown;
  } | null>;
  insertReportEvent(input: {
    reportId?: string;
    resultToken?: string;
    eventType: string;
    payload?: Record<string, unknown>;
  }): Promise<void>;
};
export async function getFullReport(
  token: string,
  deps: {
    db: FullReportDbPort;
  },
): Promise<FullReportResponse>;

3. Token Validation

If token is empty or whitespace, throw:

Invalid result token.

⸻

4. Not Found

If no report is found, throw:

Report not found.

⸻

5. Access Guard

If paymentStatus !== "PAID"

Throw:

Report is not paid.

If generationStatus !== "READY"

Throw:

Report is not ready.

If fullReport is missing or invalid

Throw:

Full report is not available.

⸻

6. Full Report Shape Validation

Validate fullReport has all 12 required section keys as non-empty strings.

Required keys:

narrativeImage
coreTemperament
sajuStructure
mbtiOverlap
mbtiConflict
loveStyle
workStudyStyle
careerDirection
moneySense
relationshipPattern
repeatingWeakness
compensationStrategy

Do not return unknown DB shape directly.

If invalid:

Full report is not available.

⸻

7. Report Viewed Event

On successful full report access, insert event:

FULL_REPORT_VIEWED

Payload:

{
  "source": "full_report_fetch"
}

Do not block report display if event insert fails.

If event insert fails, swallow/log server-side only.

8. API Route

Create:

src/app/api/reports/[token]/full/route.ts

Export:

export async function GET(
  request: Request,
  context: { params: Promise<{ token: string }> } | { params: { token: string } },
): Promise<Response>

Behavior

1. Extract token.
2. Create Supabase-backed DB adapter.
3. Call getFullReport().
4. Return full report JSON only if paid and ready.

Supabase adapter

Use:

createSupabaseServerClient()

Query reports:

id
result_token
payment_status
generation_status
full_report

where:

result_token = token

Limit 1.

Event Insert

Insert into report_events:

report_id
event_type = FULL_REPORT_VIEWED
payload

⸻

9. API Status Codes

200: success
400: invalid token
402: report is not paid
404: report not found
409: report is not ready / full report unavailable
500: unexpected error

Error response:

{
  "error": "Report is not paid."
}

Do not leak stack traces.
