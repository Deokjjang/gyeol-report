# 결리포트 Production Copy Audit

## 1. 목적

이 문서는 출시 전 화면과 정책 문구를 검토하기 위한 기준을 정리한다.

- 운영 환경에 노출될 수 있는 홈, 리포트 생성, 결과 미리보기, 정책 페이지 문구를 함께 점검한다.
- 결제, 리포트 해석, 정책 안내가 실제 상태보다 앞서 보이지 않도록 확인한다.
- 출시 전 사용자에게 오해를 줄 수 있는 결제, 리포트, 법무 관련 표현을 줄인다.

## 2. 현재 카피 전제

- 앱은 현재 출시 전 단계다.
- 리포트 생성 미리보기는 사용할 수 있다.
- 실제 결제는 비활성 상태다.
- 이용약관, 개인정보 처리방침, 환불 안내 페이지는 placeholder 성격의 초안이다.
- 리포트 내용은 자기이해를 돕는 참고 콘텐츠로 안내되어야 한다.

## 3. 공통 문구 원칙

- 신중하고 검토 가능한 표현을 사용한다.
- 결과가 정해진 것처럼 보이는 표현을 피한다.
- 결제 출시 전에는 구매가 활성화된 것처럼 보이는 문구를 쓰지 않는다.
- 문의 경로를 명확하게 제시한다.
- 한국어 문장이 모바일 화면에서도 자연스럽게 읽히도록 유지한다.

## 4. 홈 화면 카피 점검

- CTA는 리포트 미리보기 또는 리포트 생성 흐름으로 자연스럽게 이어져야 한다.
- 필요한 위치에 지원 문의 경로가 보여야 한다.
- 정책 페이지 링크가 실제 접근 가능한 경로로 연결되어야 한다.
- 유료 구매가 현재 활성화된 것처럼 보이는 표현이 없어야 한다.

## 5. 리포트 생성 화면 카피 점검

- 출생정보와 MBTI 입력 안내가 사용자가 이해하기 쉬워야 한다.
- 출생시간을 모르는 경우의 선택 또는 입력 방식이 설명되어야 한다.
- loading, success, error 상태 문구가 차분하고 실행 가능한 안내를 제공해야 한다.
- 검증 메시지는 입력값 확인과 재시도 경로를 알려야 한다.
- 개발용 전체 미리보기 또는 preview 상태 안내가 명확해야 한다.

## 6. 리포트 결과/미리보기 카피 점검

- 사주 용어는 전문성을 유지하되 읽을 수 있는 맥락을 제공해야 한다.
- MBTI는 사주 해석을 대체하지 않고 보조 비교로 제시되어야 한다.
- 고급 용어는 섹션 제목, 요약, 항목 문구를 통해 이해 가능한 단서를 제공해야 한다.
- preview 또는 gating 관련 문구가 실제 결제 기능으로 오해되지 않아야 한다.
- 리포트 해석은 자기이해용 참고자료라는 프레임을 유지해야 한다.

## 7. 정책 페이지 카피 점검

- `/terms`, `/privacy`, `/refund` 페이지는 현재 placeholder 초안임을 표시해야 한다.
- 최종 검토 전까지 출시 전 초안 안내가 유지되어야 한다.
- 문의 이메일 `official@dvem.ai`가 보이고 읽기 쉬워야 한다.

## 8. 결제 비활성 카피 점검

- 실제 결제는 활성화되어 있지 않다는 현재 상태가 유지되어야 한다.
- 카드나 결제 정보 입력을 요구하는 문구가 없어야 한다.
- 현재 구매가 가능한 것처럼 보이는 문구가 없어야 한다.
- 유료 unlock은 구현 전까지 계획, preview, gating 상태로만 설명되어야 한다.

## 9. 금지/주의 표현

- 미래 결과가 이미 정해진 것처럼 읽히는 표현을 피한다.
- 특정 성과나 관계 결과를 보증하는 표현을 피한다.
- 의료, 법률, 금융 조언처럼 보이는 표현을 피한다.
- 정책 문구가 최종 검토를 마친 것처럼 보이는 표현을 피한다.
- 결제 구현 전에는 구매 가능 상태를 암시하는 표현을 피한다.

## 10. 지원/문의 문구

- 기본 문의 이메일은 `official@dvem.ai`로 통일한다.
- 유료 출시 이후에는 리포트 생성 실패, 결제 실패, 환불 문의 경로가 명확해야 한다.
- 유료 출시 전에는 수동 지원 방식도 허용하되 사용자가 문의 경로를 찾을 수 있어야 한다.

## 11. 출시 전 수정 후보

- [ ] 홈 CTA 문구가 리포트 미리보기 흐름과 맞는지 검토한다.
- [ ] `/report/new`의 결제 및 gating 관련 문구가 현재 상태와 맞는지 검토한다.
- [ ] 정책 페이지의 placeholder 안내가 유지되는지 검토한다.
- [ ] 리포트 disclaimer가 자기이해용 참고자료라는 목적을 충분히 설명하는지 검토한다.
- [ ] 모바일 화면에서 긴 문구가 어색하게 줄바꿈되지 않는지 검토한다.
