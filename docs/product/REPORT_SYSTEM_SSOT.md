# Report System SSOT

## 1. Product Philosophy

결리포트 V2는 명리학 기반 개인화 리포트를 중심 상품으로 둔다.

핵심 철학:

- 외부 노출은 명리 중심이다.
- MBTI는 사용자가 이미 알고 있는 자기인식 언어를 보조 축으로 사용한다.
- 사주는 타고난 구조, 반복 패턴, 시간 흐름을 읽는 기준이다.
- MBTI는 사용자의 선택 방식, 관계 언어, 선호 지표, 자기보고 성향을 읽는 기준이다.
- 결리포트는 운명 단정 서비스가 아니라 자기이해와 선택을 돕는 디지털 리포트다.
- 표는 해석의 근거를 먼저 보여주는 facts layer다.
- 문장 해석은 표와 계산 근거 위에 붙는 narrative layer다.

제품 우선순위:

1. 원국 구조를 정확하고 깔끔하게 보여준다.
2. 사용자가 입력한 MBTI를 보조 성향표로 명확하게 보여준다.
3. 상품별로 필요한 추가 표를 붙인다.
4. 표 아래에서 강하고 구체적인 해석을 제공한다.
5. 공포, 예언, 진단, 보장 표현 없이 실생활 장면과 선택 기준으로 번역한다.

## 2. External Positioning And Internal Architecture

외부 포지셔닝:

- 명리 중심 개인화 리포트
- MBTI를 함께 읽는 자기이해 리포트
- 연애, 결혼, 일, 돈, 공부, 관계, 대운, 세운으로 확장되는 리포트 시스템
- 사람 상담이 아닌 자동 생성 디지털 리포트

외부 메시지 원칙:

- "사주×MBTI"를 말할 수 있지만 중심은 명리다.
- MBTI는 결론의 주체가 아니라 보조 해석 레이어다.
- 운명, 합격, 결혼, 이별, 재물, 건강을 확정하지 않는다.
- 사용자가 바로 이해할 수 있는 생활 언어를 우선한다.

내부 아키텍처:

- 명리 계산 엔진: 원국, 십성, 오행, 지장간, 신살/귀인, 합충형파해, 세운/월운 seed를 계산한다.
- MBTI 정밀 DB: `docs/product/mbti/source/*.json`의 16유형 source DB를 제품용 SSOT로 사용한다.
- bridge engine: 명리 feature와 MBTI trait을 연결해 공통점, 충돌점, 보완점을 만든다.
- table adapter: 계산/evidence/source DB를 UI 표 데이터로 변환한다.
- writer: deterministic facts를 바탕으로 상품별 문장 리포트를 작성한다.

SSOT 원칙:

- 표 데이터는 writer draft가 아니라 deterministic 계산/evidence/source adapter에서 만든다.
- writer는 표에 없는 계산값을 새로 만들지 않는다.
- 기존 `ReportOutput`은 문장 표시 결과로만 보고, V2 표 SSOT로 쓰지 않는다.
- 공통 만세력표는 `profileTable.fourPillarGrid` 중심으로 확장한다.
- MBTI 표는 MBTI source DB를 runtime adapter로 연결한다.

## 3. Input Policy

공통 입력:

- 이름
- 생년월일
- 태어난 시간
- 태어난 시간 모름 여부
- 성별
- MBTI
- 현재 직업
- 세부 직업/관심 분야
- 관계 상태

입력 정책:

- 이름은 리포트 제목과 호칭에 사용한다.
- 생년월일은 원국 계산의 필수 입력이다.
- 태어난 시간은 시주 계산에 사용한다.
- 태어난 시간을 모르는 경우 시주 컬럼은 유지하되 `모름` 또는 `-`로 표시한다.
- 성별은 대운 계산 엔진 구현 전까지 직접 대운 산출에는 사용하지 않는다.
- MBTI는 모든 V2 상품의 공통 성향표와 bridge 해석에 사용한다.
- 현재 직업은 직업, 돈, 대운, 세운 해석의 현실 장면을 고르는 데 사용한다.
- 세부 직업/관심 분야는 문장을 구체화하는 context로만 사용한다.
- 관계 상태는 연애/결혼/자녀, 궁합, 대운, 세운에서 관계 장면을 조정하는 데 사용한다.

상품별 입력 확장:

- 궁합 상품은 A/B 각각 이름, 생년월일, 태어난 시간, 성별, MBTI를 받는다.
- 궁합 상품은 관계 유형을 별도로 받는다.
- 대운 상품은 1차에서 `majorFortuneCycles` 입력 또는 외부 만세력 기준 대운표 입력이 필요하다.
- 세운 상품은 조회 연도를 받는다.

## 4. Writing Tone Policy

문장 톤:

- 강하고 구체적으로 쓴다.
- 생활 장면이 떠오르게 쓴다.
- 계산 용어를 먼저 던지지 않고, 의미를 생활 언어로 풀어준다.
- 약한 가능성 표현을 남발하지 않는다.
- "그럴 수 있습니다"만 반복하지 않는다.
- 사용자가 바로 행동 기준으로 삼을 수 있는 문장으로 끝낸다.

권장 문장:

- "이 구조는 돈과 책임이 함께 움직일 때 피로가 커집니다."
- "관계에서는 감정보다 일정과 역할 분담이 먼저 어긋날 수 있습니다."
- "올해는 새 일을 벌리는 힘보다 이미 맡은 일을 정리하는 힘이 더 중요합니다."

피해야 할 문장:

- "어쩌면 그럴 수도 있습니다."
- "좋을 수도 있고 나쁠 수도 있습니다."
- "운명적으로 정해져 있습니다."
- "반드시 큰 일이 생깁니다."

문장 밀도:

- 본문은 카드로 과분할하지 않는다.
- 큰 챕터 안에서 문단 밀도를 유지한다.
- 표, 핵심 요약, 가까운 키워드/먼 키워드는 스캔용으로 사용한다.
- 해석 본문은 충분한 길이와 장면성을 유지한다.

## 5. Safety And Forbidden Expressions

금지 원칙:

- 예언 금지
- 의학적 진단 금지
- 정신건강 진단 금지
- 법률 조언 금지
- 투자 자문 금지
- 합격, 결혼, 이별, 사고, 사망, 질병의 확정 표현 금지
- 공포 조장 금지
- 저주, 불운 확정, 위험 낙인 금지

금지 표현:

- 운명입니다
- 무조건
- 반드시
- 100%
- 확정
- 이별 확정
- 이혼 확정
- 사고수
- 단명
- 죽음
- 큰 병
- 망합니다
- 실패합니다

안전 표현 원칙:

- 단정 대신 구조와 경향으로 말한다.
- 위험 대신 관리 포인트로 번역한다.
- 사주 용어는 생활 장면으로 풀어 쓴다.
- 민감한 신살/대살은 계산 근거와 사용자 노출 정책이 확정된 경우에만 표시한다.
- diagnostic-only feature는 사용자 본문에 노출하지 않는다.

## 6. Common UI Policy

모든 상품 상단 구조:

1. 상품별 header
2. 공통 만세력표
3. 공통 MBTI 성향표
4. 상품별 추가 표
5. 상품별 본문 챕터
6. 최종 조언
7. 복사 기능

공통 UI 원칙:

- 모든 상품 상단에 만세력표를 기본 배치한다.
- 모든 상품 상단에 MBTI표를 기본 배치한다.
- 큰 챕터 단위 accordion을 사용한다.
- 본문은 카드로 과분할하지 않는다.
- 카드 안에 또 카드를 넣지 않는다.
- 표와 요약은 스캔용, 본문은 해석용으로 역할을 분리한다.
- 가까운 키워드/먼 키워드는 유지한다.
- 모바일에서 4열 만세력표와 MBTI 비교표가 읽혀야 한다.
- 표 데이터가 없으면 writer 문장으로 보완하지 않고 숨김 또는 `-`로 처리한다.

가까운 키워드/먼 키워드:

- MBTI source DB의 `closeKeywords`, `farKeywords`를 사용한다.
- 사용자에게 가까운 작동 방식과 멀거나 부담스러운 방식의 차이를 보여준다.
- 단, 낙인처럼 보이지 않게 "가까운 결", "멀게 느껴지는 결" 정도의 UI label로 둔다.

## 7. Common Table Policy

공통 표 종류:

- 만세력표
- MBTI표
- 대운표
- 세운표
- 궁합표

표 SSOT:

- 만세력표: V2 `profileTable.fourPillarGrid`
- MBTI표: `docs/product/mbti/source/*.json`
- 대운표: `majorFortuneCycles` + `MajorFortuneEvidencePacket`
- 세운표: `AnnualFortuneEvidencePacket` + `monthlyFortuneSeeds`
- 궁합표: A/B 각각의 원국/MBTI facts + compatibility evidence

표 공통 정책:

- 표는 계산/evidence 기반이다.
- 표는 LLM이 새로 쓰지 않는다.
- 표 아래 해석 본문만 writer가 작성한다.
- 오행 색상은 모든 표에서 같은 token을 쓴다.
- 천간/지지 한글 음 display dictionary는 모든 표에서 공유한다.
- 지장간, 십이운성, 십이신살, 신살/귀인, 합충형파해는 계산 가능한 항목만 노출한다.

공통 만세력표 목적:

- 모든 상품의 기본 원국 facts layer다.
- 사용자가 자신의 시주, 일주, 월주, 연주를 한 화면에서 읽게 한다.
- 표 아래 상품별 해석의 근거를 제공한다.

컬럼:

- 시주
- 일주
- 월주
- 연주

행:

- 천간
- 지지
- 지장간
- 십이운성
- 십이신살
- 신살/귀인
- 합충형파해

카드 필드:

- 한자
- 한글 음
- 십성
- 오행
- 오행 색상

오행 색상:

- 목: 초록 계열
- 화: 빨강/분홍 계열

- 출생시간 모름일 때도 시주 컬럼은 유지한다.
- 부족한 필드는 writer가 채우지 않는다.
- 모든 상품에서 같은 컴포넌트와 같은 데이터 계약을 사용한다.

## 9. MBTI Table Policy

공통 MBTI 성향표 목적:

- 사용자가 입력한 MBTI를 모든 상품에서 같은 구조로 보여준다.
- 선호 지표와 기능 서열을 표로 보여준다.
- 명리 해석과 MBTI 해석의 bridge를 보조한다.

표 블록:

- Type header
- Preference axes comparison
- Function stack table
- Core summary
- Report usage notes

선호 지표 행:

- E vs I
- S vs N
- T vs F
- J vs P

기능 서열 행:

- 주 기능
- 부 기능
- 3차 기능
- 열등 기능

SSOT:

- `docs/product/mbti/source/*.json`

source 필드:

- `type`
- `titleKo`
- `archetype`
- `oneLine`
- `preferenceAxes`
- `functionStack`
- `summary`
- `traits`
- `reportUseCases`
- `closeKeywords`
- `farKeywords`

보강 정책:

- 선호 지표의 한글명/영문명/설명은 runtime display dictionary에서 보강한다.
- 기능 code의 한글명, attitude, domain, 설명은 runtime display dictionary에서 보강한다.
- source JSON은 표 표시를 위해 수정하지 않는다.

## 10. Daeun Policy

대운 상품 정의:

- 대운 리포트는 10년 흐름과 올해 세운을 교차해서 해석하는 상품이다.
- 현재 대운이 사용자의 삶에서 어떤 장기 배경으로 작동하는지 보여준다.
- 대운 타임라인과 대운·연운 비교표를 공통 만세력표 아래에 붙인다.

1차 정책:

- 1차는 `majorFortuneCycles` 입력 기반이다.
- 외부 만세력 또는 사용자가 검증한 대운표를 입력받아 사용한다.
- 직접 대운 계산 엔진은 현재 단계에서 구현하지 않는다.
- 직접 대운 계산 엔진은 별도 spec과 parity fixture 이후 진행한다.

대운 표:

- 대운 타임라인
- 현재 대운·선택 연운 비교표

대운 타임라인:

- 10년 구간의 연도별 행
- 나이
- 현재 연도 강조
- 올해 badge
- 전환 badge
- 대운 간지
- 연운 간지
- 대운/연운 십성
- 오행 색상

대운·연운 비교표:

## 11. Saeun Policy

세운 상품 정의:

- 세운 상품은 한 해의 연운과 월운 흐름을 보는 상품이다.
- 대운 정보가 있으면 대운·연운 비교표를 함께 제공한다.
- 월운은 상반기와 하반기로 나누어 표로 보여준다.

조회 정책:

- 기본 조회 범위는 과거 5년 + 올해다.
- 매년 12월부터 다음 해 세운을 오픈한다.
- 다음 해 세운은 신년사주 성격으로 제공한다.
- 과거 10년 조회는 2차 확장 후보로 둔다.

1차 계산 정책:

- 기존 annual fortune evidence 구조를 활용한다.
- 연운은 `AnnualFortuneEvidencePacket.annualGanji`를 기준으로 한다.
- 월운은 `monthlyFortuneSeeds`를 기준으로 한다.
- 월운은 `달력월 기준 운영 가이드`로 표시한다.
- 절기 기준 정밀 월운은 별도 spec로 분리한다.

세운 표:

- 대운·연운 비교표
- 월운 상반기 표
- 월운 하반기 표

월운 표:

- 상반기: 1월~6월
- 하반기: 7월~12월
- 각 월의 천간/지지 카드
- 월운 간지
- 십성
- 오행 색상
- 지장간
- 십이운성
- 십이신살
- 신살/귀인
- 합충형파해
- 핵심 관계/주의 label

## 12. Compatibility Policy

궁합 상품 정의:

- 궁합은 연애/결혼 전용이 아니다.
- 궁합은 관계 궁합 전체를 다루는 상품군이다.
- 두 사람의 명리 구조, MBTI 자기인식, 관계 목적을 함께 본다.
- 점수는 재미와 비교를 위한 요약값이지 성공/실패 판정이 아니다.

포함 카테고리:

- 연애 궁합
- 결혼 궁합
- 부모·자식 궁합
- 직장 동료 궁합
- 상사·부하 궁합
- 사업/협업 궁합
- 친구/인간관계 궁합

관계 유형 운영:

- 연애와 결혼은 독립 카테고리로 둔다.
- 부모·자식은 가족 궁합 안에서 우선 지원하고, 필요 시 2차에서 세분화한다.
- 직장 동료, 상사·부하, 사업/협업은 업무/협업 궁합 안에서 우선 지원하고, 필요 시 2차에서 세분화한다.
- 친구/인간관계는 friendship/general relationship으로 둔다.

궁합 상단 표 구조:

1. A 만세력표
2. A MBTI표
3. 중간 연결/궁합 요약표
4. B 만세력표
5. B MBTI표

중간 연결/궁합 요약표:

- 관계 유형
- 총점과 주의 문구
- 끌림/소통/생활 리듬/갈등 회복/장기 안정성/성장 보완 breakdown
- 명리 연결 요약
- MBTI 연결 요약
- 좋은 작동 방식
- 충돌이 생기는 방식
- 오늘부터 쓸 관계 규칙

궁합 문장 정책:

- 관계 유형별 언어를 구분한다.
- 연애/썸에는 업무 용어를 쓰지 않는다.
- 업무/협업에는 로맨스 용어를 쓰지 않는다.
- 가족 궁합에는 가족, 생활, 정서, 역할 경계 언어를 쓴다.
- MBTI 후보 추천은 하지 않는다.
- 두 사람의 실제 입력값과 evidence에 없는 사주 용어를 만들지 않는다.

## 13. Product Calculation Policy

### 종합

- 공통 만세력표와 공통 MBTI표를 상단에 둔다.
- 명리 구조를 중심으로 전체 성향, 일, 돈, 공부, 관계, 성장 포인트를 해석한다.
- MBTI는 보조 성향표와 bridge 해석으로 사용한다.
- V2 `profileTable`과 chapter 구조를 기준으로 한다.

### 직업·커리어·돈·학업

- 명리의 십성, 오행, 일간, 월지, 식상/재성/관성/인성 흐름을 중심으로 한다.
- MBTI traits의 `career`, `workplace`, `money`, `investment`, `study`를 보조한다.
- 직업명 단정이나 투자 조언을 하지 않는다.
- 적합한 환경, 피해야 할 운영 방식, 돈 관리 습관, 공부 루틴으로 번역한다.

### 연애·결혼·자녀

- 명리의 일지, 십성, 관계 신호, 표현 방식, 책임 방식, 생활 리듬을 중심으로 한다.
- MBTI traits의 `love`, `marriage`, `parenting`, `child`, `relationships`, `communication`을 보조한다.
- 결혼 시기, 이별, 출산, 가족 사건을 확정하지 않는다.
- 관계 장면, 말투, 경계, 역할 분담, 회복 방식으로 번역한다.

### 궁합

- A/B 각각의 원국과 MBTI를 먼저 독립적으로 보여준다.
- 중간 연결/궁합 요약표를 통해 두 사람의 관계 구조를 보여준다.
- 관계 유형별로 점수 label과 조언 언어를 다르게 적용한다.
- 궁합은 성공/실패 판정이 아니라 조율 구조다.

### 대운

- 10년 대운 흐름과 올해 세운을 교차 해석한다.
- 1차는 `majorFortuneCycles` 입력 기반이다.
- 대운 타임라인과 대운·연운 비교표를 상단 표로 제공한다.
- 직접 대운 계산 엔진은 별도 spec/parity 이후 진행한다.

### 세운

- 선택 연도 연운과 월운을 해석한다.
- 기본 조회 범위는 과거 5년 + 올해다.
- 12월부터 다음 해 세운을 신년사주 성격으로 연다.
- 월운은 상반기/하반기 표로 보여준다.
- 절기 기준 정밀 월운은 별도 고도화로 분리한다.

## 14. Copy And AI Summary Policy

복사 기능 종류:

- 전체 리포트 복사
- AI용 요약본 복사

전체 리포트 복사:

- 사용자가 화면에서 읽는 유료 리포트 본문을 복사한다.
- 상단 기본 입력, 주요 표 요약, 본문 챕터, 최종 조언을 포함한다.
- 내부 debug, evidence id, validator warning, payment id는 포함하지 않는다.
- 개인정보 노출을 최소화한다.

AI용 요약본:

- 사용자가 다른 AI 도구에 붙여 넣기 좋은 압축 요약본이다.
- 긴 본문 전체가 아니라 구조화된 요약을 제공한다.
- 포함 항목은 입력 요약, 원국 핵심, MBTI 핵심, 주요 강점, 주의점, 관계/일/돈/시간 흐름 핵심이다.
- 민감한 결제 정보, access token, 내부 source id는 포함하지 않는다.
- 예언, 진단, 보장 표현 없이 "참고용 자기이해 요약"임을 유지한다.

복사 UI 정책:

- 복사 버튼은 리포트 상단 또는 하단에 둔다.
- 복사 성공/실패 상태를 명확히 보여준다.
- 모바일에서 버튼이 본문을 가리지 않게 한다.

## 15. Product Card Version Policy

상품 카드 정책:

- 상품 카드는 제품 버전 badge를 표시한다.
- 현재 카드의 `versionBadgeKo`는 사용자에게 보이는 상품 표면 버전이다.
- 계산 엔진 버전, writer 버전, snapshot version과 혼동하지 않는다.
- 출시 준비 중 상품은 구매 가능 상태로 보이지 않게 한다.
- 구매 가능 상품은 가격, 런칭가, 정가, delivery type, 자동 생성 리포트 고지를 함께 보여준다.

버전 정책:

- 종합 리포트: `v1.0` 카드 badge를 유지하되 내부 draft는 V2 구조를 사용할 수 있다.
- 궁합, 대운, 세운은 출시 전까지 `출시 준비 중` 상태를 유지한다.
- 새 표/계산 정책이 추가되어도 상품 카드 버전은 사용자에게 의미 있는 출시 단위에서만 올린다.
- 내부 schema version 변경은 별도 migration/snapshot 정책으로 관리한다.

상품 카드 문구:

- 사람 상담이 아닌 자동 생성 디지털 리포트임을 명시한다.
- 의료, 법률, 투자 자문이 아님을 명시한다.
- 결과물은 자기이해와 참고 목적의 정보임을 명시한다.

## 16. Development Order

추천 개발 순서:

1. 공통 display dictionary를 만든다.
2. 오행 색상 token을 공통화한다.
3. 천간/지지 한글 음 mapping을 공통화한다.
4. 공통 만세력표 data adapter를 만든다.
5. 공통 만세력표 UI 컴포넌트를 만든다.
6. MBTI source DB runtime adapter를 만든다.
7. 공통 MBTI 성향표 UI 컴포넌트를 만든다.
8. 종합 리포트 상단에 만세력표와 MBTI표를 연결한다.
9. 가까운 키워드/먼 키워드 표시를 연결한다.
10. 전체 리포트 복사와 AI용 요약본 복사 정책을 구현한다.
11. 대운 table data contract를 만든다.
12. 대운 타임라인과 대운·연운 비교표 adapter를 만든다.
13. 대운 상품 UI를 연결한다.
14. 세운 table data contract를 만든다.
15. 연운 표와 월운 상반기/하반기 adapter를 만든다.
16. 세운 상품 UI를 연결한다.
17. 궁합 상단 표 구조를 만든다.
18. 궁합 관계 카테고리를 제품 정책에 맞게 세분화한다.
19. 상품별 writer가 표 데이터를 변경하지 못하게 validator를 보강한다.
20. fixture와 visual QA로 모든 표의 모바일 표시를 검증한다.
21. 직접 대운 계산 엔진은 별도 spec/parity fixture 이후 진행한다.
22. 절기 기준 정밀 월운은 별도 spec/fixture 이후 진행한다.
